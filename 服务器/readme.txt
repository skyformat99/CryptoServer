服务器端。
架构模式：
两套套接字。
首先使用LOG_PORT进行监听链接登录信息，并转发它到每个客户端。
开启第二线程，使用MSG_PORT监听聊天消息，然后使用LOG_port转发，然后再绑定回MSG_PORT继续监听。
环境配置：
首先下载CRYPTO++ ，
点击生成，会产生一个LIB文件（也许还有dll或者.h？）。放到项目里面去，然后在项目属性-》C++-》包含目录里面添加.h，在链接器-》常规-》库目录里添加lib。如果有其他链接文件同理添加。
第二，把pthread文件夹放在某地，同上添加方式。
第三，把属性-》代码生成-》调试方式 改为MTD。
第四，把msado15这两个文件放在文件夹里面，报错不要紧，重新生成即可。
第五，把pthreadVC2.dll放在文件夹下。

数据库建库：
数据库名 test
表名  table1
列名 num1 num2 
非空，10位 nchar。

num1 是账号，num2是密码

字符串比较函数没写好。
所以，数据库建库的时候，m密码只设一位字符，否则会出错。

本地文件有签名。
